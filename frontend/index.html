Catching throttled brute-force

The evasion
Password tools throttle to ~1 attempt every 2 s — invisible to the classic 2-second count window.
Example
20 SSH login attempts, one every ~2.2s — far under any per-2s connection-count alarm.
The feature
auth_bruteforce_score — attempts per source hitting ftp/21, ssh/22, telnet/23 in a rolling 60-second window.
Retrained on it
The Isolation Forest was retrained with this feature so throttled brute-force stands out statistically too.
The rule on top
score ≥ 15 force-flags the connection regardless of the model's verdict — rules layered on ML, like real security tools.
Why layer rules
ML catches statistical anomalies; a deterministic rule guarantees this well-understood attack is never missed.
Live so far this session: 0 auth-port (ftp/ssh/telnet) connections observed · highest attempt count seen: 0 in a 60s window · 0 force-flagged by the ≥15 rule.
