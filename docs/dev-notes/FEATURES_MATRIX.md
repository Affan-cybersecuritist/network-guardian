## ❌ WHAT YOUR PROJECT CANNOT DO (5% Unsolvable)

### Category 1: Physical Security
| Limitation | Reason | Workaround |
|-----------|--------|-----------|
| Detect physical theft | Out of scope (network IDS) | CCTV cameras, access control |
| Prevent hardware tampering | Happens before deployment | Supply chain monitoring |
| Detect evil maid attacks | BIOS/firmware level | TPM, firmware verification |

### Category 2: Cryptography
| Limitation | Reason | Workaround |
|-----------|--------|-----------|
| Break encryption | Mathematically impossible | Post-quantum crypto (future) |
| Decrypt old traffic | No decryption key available | Capture key during attack |
| Defend against quantum | Quantum computers don't exist yet | Post-quantum algorithms |

### Category 3: Social Engineering
| Limitation | Reason | Workaround |
|-----------|--------|-----------|
| Stop phishing clicks | User choice, not tech flaw | Security awareness training |
| Prevent password sharing | Would require mind-reading | User behavior training |
| Detect social calls | Telephone outside network | Phone security, training |

### Category 4: Air-Gapped Networks
| Limitation | Reason | Workaround |
|-----------|--------|-----------|
| Monitor disconnected networks | No network access | Deploy local agents |
| Detect USB exfiltration | Physical media is offline | USB blocking, DLP |
| Track data leaving network | Visibility ends at boundary | Endpoint agents, watermarking |

### Category 5: Supply Chain
| Limitation | Reason | Workaround |
|-----------|--------|-----------|
| Scan code vulnerabilities | SAST required (not IDS scope) | SonarQube, Checkmarx |
| Verify third-party libs | Requires binary analysis | Software Composition Analysis |
| Catch zero-day in frameworks | No signature until disclosure | Behavioral detection, sandboxing |
| Prevent supply chain compromise | Happens before deployment | Vendor vetting, code signing |

### Category 6: Application Logic
| Limitation | Reason | Workaround |
|-----------|--------|-----------|

| Detect race conditions | Requires code-level analysis | Static analysis, fuzzing |
